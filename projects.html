Monotone Rules Language & CALM Convergence Demo

Design and implement a small declarative rules language, influenced by Bloom, restricted so facts are only ever accumulated - never retracted. Programs execute across multiple simulated nodes that exchange derived facts in arbitrary orders; the artifact illustrates the CALM perspective that such monotonic computations converge to the same outcome regardless of delivery order, whereas optional non-monotonic extensions can diverge without coordination.

The surface syntax is a rule set expressed as a DSL (along the lines of treating SQL as a language interpreted by an engine, not executed directly by the CPU). A runtime written in a general-purpose language parses rule files according to fixed lexical and grammatical rules and drives execution: for an MVP confined to pure monotonic rules, execution reduces to maintaining and growing a store of facts.

The demonstration is empirical equivalence under reordering: given two nodes fed the same logical inputs in different network orders, both settle on multisets that collapse to the same set once duplicates are removed and order is ignored - for example, {a, a, b, c}, {a, b, c}, and {c, b, a} represent the same fact basis. That path avoids the heavier alternative of a bespoke compiled binary, dedicated bytecode VM, and orchestrated cluster purely to stage the same convergence argument.

Custom Block Cipher

Design and implement a custom symmetric block cipher whose structure draws on ideas visible in classics such as Blowfish: fixed block width, iterated rounds built from nonlinear mixing, XOR-layered partial substitutions, and permutation-style diffusion so that lightweight rounds still compose into something nontrivial when exercised end to end.

The engineering spine is modular and extensible: a dedicated key schedule expands user material into round-specific subkeys; round transformations stay isolated behind narrow interfaces so alternate S-box layouts, tweak counts, or round counts can be experimented with without rewriting the whole pipeline.

Above the primitive sits framing work - correct padding for arbitrary-length plaintext and wired-up modes of operation so the same core block function can run under ECB-style isolation versus chaining IV-dependent modes such as CBC. Pair that with unit tests, documented invariants (block boundaries, decryption round-trip), and repeatable benchmarks or fuzz-adjacent checks where helpful to confirm round-trip fidelity rather than cryptographic strength.
